# 🧪 SARAS Validation & Testing Methodology

This document outlines the testing strategies and validation methods employed to ensure the accuracy, reliability, and academic rigor of the SARAS platform.

## 1. Statistical Engine (SIGMA) Validation

Given that SIGMA auto-generates narratives for Chapter IV (BAB IV) of academic theses, mathematical precision is non-negotiable.

### 1.1 Benchmark Testing against R
- **Methodology:** We utilize a suite of 50 standardized datasets covering common academic research scenarios (marketing, psychology, engineering).
- **Process:** The OLS regression outputs generated by SIGMA's Go backend (using `gonum`) are programmatically compared against the outputs generated by R's `lm()` function.
- **Success Criteria:** 
  - Coefficients (B) must match R up to 4 decimal places.
  - Standard Errors, t-values, and p-values must match up to 4 decimal places.
  - F-Statistics and R-Squared values must match exactly.

### 1.2 Anti-Hallucination Measures
- **System Prompt Constraints:** The Gemini API is constrained by rigid system instructions forcing it to rely *only* on the provided mathematical matrix.
- **Fail-Safe Mechanism:** If $p > 0.05$, the AI is explicitly instructed to refuse to conclude any significant correlation or influence, preventing the hallucination of positive results.

## 2. Integrity Detection (ARIA) Validation

### 2.1 Edge-Case Synthetic Datasets
We employ synthetic datasets containing known flaws to validate the ARIA module:
- **Z-Score Outliers:** Datasets injected with values $> 4\sigma$ from the mean.
- **Benford's Law Deviations:** Datasets generated via RNG vs. naturally occurring datasets, ensuring ARIA flags the RNG data.
- **Duplication Injection:** Datasets with deliberately duplicated rows (exact and near-match) to test the duplicate detection algorithm.

## 3. Security & PII Scrubbing Validation

### 3.1 PII Payload Testing
- The `internal/gemini/scrubber.go` is tested using unit tests covering various formats of Indonesian phone numbers (`+62`, `08`, `628`), student ID numbers (NIMs of various university formats), and email addresses.
- **Assertion:** The outgoing prompt string must contain strictly 0 instances of the injected PII before being dispatched to the Gemini endpoint.

### 1.3 Mixed-Dataset Robustness Testing
- **Methodology:** We test ingestion of CSV files containing both numeric variables and non-numeric variables (e.g., student IDs, string category descriptions, timestamps).
- **Assertion:** The Go backend regression engine must successfully unmarshal requests into interface maps, skip non-numeric variables, convert valid numeric strings or numeric formats dynamically, and run regression computations without returning `Invalid request payload` or panicking.
